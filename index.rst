Presentazione
-------------

Agli albori dell'era informatica (fino alla fine degli anni '80) si studiavano il Linguaggio C e il FORTRAN come al liceo classico si studiavano il greco e il latino.

Era impensabile che un amministratore di sistema Unix non conoscesse il C, così come un fisico non conoscesse la matematica applicata.
Per parlare del funzionamento di una _feature_ di Unix si scriveva semplicemente il codice C implementativi.

Ora i cultori del Linguaggio C sono guardati con sospetto dagli ufficiali di sicurezza che compiono un profilo psicologico.

In Linguaggio C infatti si scrivono:

* device drivers
* applicativi per sistemi Real Time
* programmi per hacker

Se le attività di una persona non rientrano nelle prime due categorie, la persona è uno hacker potenziale.

Infatti per le prime due categorie il Linguaggio C, a volte coadiuvato da costrutti sintattici del C++, è ancora indispensabile.

E' un linguaggio che permette di lavorare molto vicino agli aspetti fisici della macchina, pur mantenendo un minimo livello di astrazione.

L'assembler sarebbe meglio, se non fosse per due problemi:

* CPU diverse hanno diversi set di istruzioni assembler
* la curva di apprendimento dell'assembler è ancora più ripida di quella del linguaggio C

Quindi il C non è morto, anzi è in continuo sviluppo.

Che non si può dire però del greco antico.
